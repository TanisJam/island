# Pilares de diseño

> Fuente: GDD v2 §4.

Los siete pilares que toda mecánica nueva debe respetar. Si una feature no refuerza
al menos uno, probablemente no pertenece al juego (ver
[reglas de expansión](../2-dominio/reglas-de-expansion.md)).

## 4.1. El mundo se entiende observándolo

El primer contacto con un objeto no debe ser explotarlo, sino observarlo. El juego
debe incentivar: **mirar, examinar, comparar, probar, deducir**. Cada tile,
recurso, herramienta o máquina puede contar algo sobre el mundo.

## 4.2. El crafting es físico

Los objetos no son números abstractos:

- Una piedra es una piedra.
- Una rama ocupa espacio.
- Un tronco se lleva con ambas manos.
- Una pila de piedras existe en el mundo.
- Un contenedor tiene una grilla física.
- El campamento funciona como inventario expandido.

## 4.3. Las manos definen la intención

El jugador no selecciona acciones desde una barra genérica. Las acciones
disponibles dependen principalmente de:

- qué tiene en la mano izquierda;
- qué tiene en la mano derecha;
- qué objeto o tile está tocando;
- qué sabe el personaje;
- qué herramientas o técnicas posee;
- qué distancia hay al objetivo.

## 4.4. El conocimiento es progreso

No alcanza con tener materiales: el personaje también necesita aprender técnicas
(fundición básica, carpintería, molienda, trenzado, contención de calor, triturado
fino, ensamblaje rudimentario...).

Dos jugadores pueden tener los mismos objetos pero distintas posibilidades si
conocen técnicas diferentes.

## 4.5. Las máquinas nacen de dolores

Las máquinas no aparecen porque sí. Aparecen porque el jugador hace algo manual
demasiadas veces o se encuentra con un cuello de botella.

- Machacar granos a mano muchas veces puede desbloquear la idea de un molino.
- Romper piedras manualmente muchas veces puede desbloquear la idea de un triturador.
- Intentar mantener fuego estable puede desbloquear el horno.

## 4.6. La isla bloquea y guía orgánicamente

El jugador empieza en una zona limitada. No necesita una pared invisible. El bloqueo
natural puede ser: jungla muy espesa, árboles imposibles de talar con las manos,
rocas grandes, ríos, acantilados, pantanos, mar abierto.

La progresión abre caminos: hacha para despejar árboles, pico para romper rocas,
cuerda para bajar desniveles, balsa para cruzar agua, puente para conectar zonas,
antorcha para explorar cuevas.

## 4.7. La cooperación acelera, pero no reemplaza

El multiplayer asincrónico permite visitar bases, usar máquinas públicas, aprender
técnicas, enseñar técnicas, intercambiar recursos, dejar objetos y colaborar en
proyectos comunes.

Pero otro jugador no debería poder simplemente regalar todo el progreso. Puede
acelerar, orientar o prestar infraestructura.
