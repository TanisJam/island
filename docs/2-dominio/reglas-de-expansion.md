# Reglas de diseño para expansión

> Fuente: GDD v2 §31 y §32.

Cómo decidir si una mecánica nueva pertenece al juego, y qué preguntas hacerse al
agregar contenido. Es el filtro que mantiene la coherencia con los
[pilares](../0-vision/02-pilares.md).

## 31. Cada sistema nuevo debe cumplir al menos una función

```
1. Abrir una forma nueva de interactuar con el mundo.
2. Resolver un dolor repetitivo.
3. Crear una decisión de organización física.
4. Hacer útil una técnica.
5. Hacer útil a otro jugador.
6. Abrir una nueva zona.
7. Generar una nueva pregunta interesante.
```

Si una mecánica sólo agrega números, probablemente no pertenece.

## 32. Preguntas guía para diseñar contenido

**Cuando se agregue un nuevo recurso:**

```
¿Qué se puede hacer con él en la mano?
¿Qué pasa si se calienta?
¿Qué pasa si se moja?
¿Qué pasa si se tritura?
¿Qué máquina lo procesa?
¿Qué técnica lo entiende mejor?
¿Qué bloqueo ayuda a resolver?
```

**Cuando se agregue una nueva máquina:**

```
¿Qué dolor manual resuelve?
¿Qué recursos transforma?
¿Qué necesita para funcionar?
¿Qué produce como residuo?
¿Qué mejora permite?
¿Qué interacción social habilita?
```

**Cuando se agregue una nueva técnica:**

```
¿Qué permite ver o entender?
¿Qué acciones desbloquea?
¿Cómo se aprende por práctica?
¿Cómo se enseña a otro?
¿Qué máquinas o herramientas mejora?
```

## Regla de oro del MVP

> Fuente: MVP Spec §29. Aplica al alcance del primer slice.

Cada cosa implementada debe reforzar al menos una de estas ideas:

```
Estoy en un mundo físico.
Mis manos importan.
Mirar me ayuda a entender.
Los objetos tienen forma y peso.
El campamento es parte del inventario.
El descubrimiento nace de probar con sentido.
Las herramientas cambian mi relación con la isla.
La isla se abre de manera orgánica.
```

Si una feature no refuerza ninguna de esas ideas, no pertenece al MVP.
